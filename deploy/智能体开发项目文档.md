### 1.2.1 关于模型配置

```bash
1. 有关项目的环境变量(主要有各模型的api_key) 统一放到工程的.env 文件中，注意，在部署时该文件需要手动更新(或可以将文件从.gitignore移出)
2. 项目框架支持配置多个模型，各模型对应的ChatModelProvider集中在 src/config/static/models.py 文件中
   目前主要是用 
   "deepseek": ChatModelProvider(
        name="DeepSeek",
        url="https://platform.deepseek.com/api-docs/zh-cn/pricing",
        base_url="https://api.deepseek.com/v1",
        default="deepseek-chat",
        env="DEEPSEEK_API_KEY",
        models=["deepseek-chat", "deepseek-reasoner"],
    )
   default_model即默认模型在 src/config/app.py 指定
   default_model: str = Field(
        default="deepseek/deepseek-chat",
        description="默认对话模型",
    )



#注意docker-compose.yml在工程根目录下，其中只开放了必要的服务,如果要运行全部服务，就完全放开所有注释

```


### 1.2.2 关于智能体(agent)

```bash

项目中的智能体代码路径为src/agents/ 该目录下的各智能体在项目启动时自动初始化
1. carbot_master_agent 为目前业务开发的主要多任务智能体，它是由deepseek+sub-agent设计，实现代码在carbot_master_agent/graph.py文件中
2. 各子智能体相关的tools代码，也在carbot_master_agent目录下,根据功能进行汇集在各 xxxx_tools.py文件中
3. carbot_master_agent中子智能体的任务分配以及执行引导的 system_prompt 在carbot_master_agent/prompts.py中定义 
4. 目前与业务相关的子智能体主要有
                        kb_sub_agent,---> 知识库智能体
                        car_home_sub_agent,  ---> 汽车之家智能体
                        xhs_content_generate_sub_agent,   ---> 小红书风格内容创作智能体
                        digital_human_video_sub_agent,   ---> 数字人生成智能体
                        video_analysis_report_sub_agent    ----> （汽车之家）社会事件视频分析及报告知能体(未做完)



```
## 1.2.3 关于智能体接口


```bash

根据 Yuxi-Know框架开发规范，智能体服务接口采用fastAPI设计，流式数据输出(sse),
代码在 server/routers/chat_router.py 中 
@chat.post("/agent/{agent_id}")
async def chat_agent(
    agent_id: str,
    query: str = Body(...),
    config: dict = Body({}),
    meta: dict = Body({}),
    image_content: str | None = Body(None),
    # current_user: User = Depends(get_required_user), ---->这里注释了用户信息,不需要用户登录的token即可访问，
    db: AsyncSession = Depends(get_db),
):
...
chat_agent 为主要对外服务接口

这里 agent_id为 CarbotMasterAgent 在 src/agents/carbot_master_agent/_init_.py中定义
比如，用我们多任务对话智能体(CarbotMasterAgent)就访问 
http://0.0.0.0:5050/api/chat/agent/CarbotMasterAgent

更多接口访问的例子可参考 工程中 docs/command.txt文件



```
